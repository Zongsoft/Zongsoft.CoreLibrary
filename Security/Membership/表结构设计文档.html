其它说明·Remark
假设有角色 Market 和用户 Jack，其中 Jack 是 Market 的角色成员之一。在 Permission 授权表有记录如下：
MemberName MemberType SchemaId ActionId Protect
Market 1 SalesOrder Select 1(Grant)
... ... ... ... ...
其表示 Market 角色下属所有成员默认拥有对 SalesOrder（销售订单）应用模块授权有 Select（读取）业务数据的权力。且在 PermissionFilter 授权过滤表中有记录如下：
MemberName MemberType SchemaId ActionId Filter
Jack 0 SalesOrder Select TSalesOrder.CustomerNo
Jack 0 SalesOrder Select TSalesOrderDetail.Price
Jack 0 SalesOrder Select TSalesOrderDetail.Amount
Jack 0 SalesOrder Select TSalesOrderDetail.Quantity
... ... ... ... ...
以上两个授权表中的数据指示出 Jack 默认拥有 Select（读取）SalesOrder（销售订单）应用模块中所有业务数据的权力（继承自 Market 角色的授权），但是他却被拒绝 Select（读取）SalesOrder（销售订单）应用模块中如下字段的数据：
  • TSalesOrder.CustomerNo（销售订单所属的客户编号）
  • TSalesOrderDetail.Price（产品单价）
  • TSalesOrderDetail.Amount（销售金额）
  • TSalesOrderDetail.Quantity（销售数量）

以上过滤器语法可以简写成一行：TSalesOrder.CustomerNo; TSalesOrderDetail.Price, Amount, Quantity

注意：在 PermissionFilter 表中的记录始终表示的是拒绝授权的信息。
